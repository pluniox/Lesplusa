## Méthode de travail
On a travaillé en parallèle pour avancer plus vite.

- Chaque personne prenait une fonctionnalité précise.
- On développait chacun de notre côté en classe, en testant localement.
- Dès qu’une fonctionnalité était prête, on envoyait le code à la personne qui centralise.
- La centralisation se faisait en local : récupération du code de chacun.
- Quand tout était stable, on faisait un push global sur GitHub pour garder un historique propre et cohérent.
